                 Notation conventions for TESTCOB subcommands

 1. The symbols below are used to define subcommand format. With the exception
 of the hyphen, which may be used as a continuation character under TSO, these
 symbols should not be typed at the terminal.

          NAME                  SYMBOL                 MEANING

       hyphen                     -            joins lower case
                                               letters, words, and
                                               symbols to form a single
                                               variable

       underscore               ______         indicates a default
                                               option, one you need not
                                               type if it is the one
                                               you want

       slash                      //           group related items: you
                                               MAY choose one of the
                                               items between slashes

       exclamation point         !!            group related items: you
                                               MUST choose one of the
                                               items between exclamation
                                               marks

       ellipsis                   ...          indicates that the
                                               preceding item or group
                                               of items can be repeated
                                               one or more times in
                                               succession if you want

 2. Upper case letters, numbers, and symbols other than the ones above must be
 typed exactly as they appear in the format.

 3. Lower case letters represent variable information that is supplied by the
 user.
1




                 Notation conventions for TESTCOB subcommands

                                 (continued)

 4. The following terms have special meanings in subcommand formats:

    * addr         is a COBOL index-name, a COBOL data-name, or a COBOL
                   special register
    * expression   is a relationship expressed in this format:

                   addr-1  operator   !value! !addr-2!

                   (Addr, operator, and value are defined here.)
    * operator     is one of the following:

                   EQ or =   equal to
                   GT or >   greater than
                   LT or <   less than
                   NE or ¬=  not equal to
                   GE or >=  greater than or equal to
                   LE or <=  less than or equal to

    * statement    is expressed: line number /.verb number/
    * symbol       is any character combination that follows the rules
                   for forming a data-name in the COBOL language.
    * value        is a literal constant or a figurative constant.

 5. A range of statements, which includes all statements within the range, is
 delimited by two statements separated by a colon:
                   statement-1:statement-2
    (this applies to a range of data-names or index-names.)
 6. A list of statements, which includes only the statements listed, is
 enclosed in parentheses, and each statement is separated from the next by
 commas, blanks, or tabs:
                   (statement-1,statement-2,statement-3)
    (this applies to a list of addrs, symbols, and so on.)
 7. Valid delimiters between operands in all TESTCOB subcommands are blanks,
 commas, and tabs. You can use these interchangeably.

 Subcommand/Operands

 AT  !statement!
     !(statement-1/,statement-2/...)!
     !statement-3:statement-4!

     /(subcommand list)/ /COUNT(n)/

     /NOTIFY/
+     ______
     /NONOTIFY/

     statement
        is the point where you want to establish an AT breakpoint.

     (statement-1,statement-2)
        are points where you want to establish AT breakpoints. Use this operand
        if you want to establish AT breakpoints in several NONCONSECUTIVE
        statements in your program.

     statement-3:statement-4
        delimit a range of statements where you want to establish AT
        breakpoints. Using this operand establishes AT breakpoints at every
        statement, which means at every COBOL verb, between statement-3 and
        statement-4. Therefore, statement-3 must appear before statement-4 in
        the program, and both statements must appear in the same program.

     (subcommand list)
        is a list of one or more TESTCOB subcommands. Each subcommand in the
        list must be separated from the next by a semicolon.  Also, the list
        may include AT if you wish. When an AT breakpoint is executed, the
        system stops program execution and then executes the subcommand list if
        there is one. Once the subcommand list is executed, control is returned
        to the terminal where you may enter additional subcommands or cause
        program execution to resume.

        NOTE: If the Interactive Debug command processor encounters the GO or
        RUN subcommand in the subcommand list, no additional subcommands will
        be executed after GO or RUN. In this situation, program execution is
        resumed. If the END subcommand is encountered in the subcommand list,
        the command processor immediately terminates the debugging session.

     COUNT(n)
        is a way to specify that you want the AT breakpoint to cause an
        interruption every nth time it is encountered during program execution,
        where n is the value of a positive integer.  (Thus, if COUNT(5) is
        specified, the FIRST execution of the AT breakpoint occurs the FIFTH
        time the breakpoint is encountered. The verb at this breakpoint is
        executed FOUR times, and the interruption occurs before the verb is
        executed a FIFTH time.)

     NOTIFY
        means that you want the system to print an identifying message every
        time the AT breakpoint is executed. To suppress the message, specify
        NONOTIFY. NOTIFY is the default.

 EQUATE (or EQ)

        symbol

        !data-name!
        !index-name!
        !special register!

     symbol
        is any character conbination that follows the rules for forming a COBOL
        data-name. It cannot be qualified nor used as a qualifier. NOTE: A
        symbol cannot be qualified by program-name.

        A symbol can represent only one data-name, index-name, or COBOL special
        register. If the same symbol is used in a second EQUATE subcommand, the
        second definition is the only one that the system will recognize and
        accept. Once a symbol has been defined by an EQUATE subcommand, it is
        entered into a symbol table. Thereafter, the symbol can be used in any
        subcommand in place of the data-name, index-name, or special register
        it represents. The definition is effective until you remove the symbol
        from the symbol table by using the DROP or RUN subcommand or until you
        end the debugging session, whichever comes first. If the symbol is
        defined already in the program, the symbol defined in EQUATE takes
        priority.

     data-name
        is any data-name in your program, including a qualified data-name, such
        as A of B of C.

     index-name
        is any index-name in your program.

     special register
        is any one of the COBOL special registers.

           addr-1   operator  !value! !addr-2!

        padding, will be the same as those in the COBOL language.

        the terminal if the expression is true.

        indicates that you want to restart program execution if the
