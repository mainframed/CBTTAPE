Use of TSO transmit/receive in batch
The following article describes how to use the TSO transmit and receive commands
in batch, thus making it possible to automate file transfer between
installations without having to get a TSO user to execute the commands.
Required software - MVS/370 SP 1.3.3, JES2 1.3.4, TSO/E 1.2.0.  The concerned
functions should be applicable to other releases of the same software as well as
MVS/XA.
REQUIREMENTS
The following requirements are necessary to make the process work:
1    Both sender and receiver must have customised their spool systems to
support NJE and defined each other to their JES.
2    Macro INMXPARM set up and customised.
3    Connection between the spool systems must have been established from either
end.
4    To receive in batch, the RECEIVE command must be issued with the parameter
USERID(user-id name); this requires the ‘operator’ attribute which is not
directly available in background.  To support this, the TSO command TSOOPER has
been written to set the operator attribute in the PSCB.
5    TSOOPER must execute in key zero to modify the PSCB, and therefore must be
authorised.  Authorisation can be accomplished via a link with AC=1 into an
authorised library together with the command mentioned in table IKJEFTE2 when
the program should execute under TSO.  Alternatively, authorise the program
dynamically via an installation-written authorisation SVC, if such an SVC
exists.
6    TSO commands TRANSMIT, XMIT, and RECEIVE must all be authorised in table
IKJEFTE2.

TSOOPER
* TSO COMMAND TO GET OPERATOR ATTRIBUTE USED BY RECEIVER IN BACKGROUND
* MUST BE LINKED WITH AC=1 INTO AN AUTHORISED LIBRARY AND THEN ALSO
* PUT INTO TABLE IKJEFTE2, OR ALTERNATIVELY:
* ISSUE CALL TO INSTALLATION AUTHORISATION SVC IF SUCH EXISTS:
* IN THIS SITE CALLED VIA MACRO AUTHON AND DISABLED VIA AUTHOFF.
*  this program contains user macros initr, exitr, authon and authoff
         IHAPSA
         IKJTCB
         IEZJSCB
         IKJPSCB
TSOOPER  INITR                            see issue 15 pages 20-24
         USING PSA,R0                     DUMMY USING FOR PSA
         L     R10,PSATOLD                GET TCB-ADDRESS
         USING TCB,R10                    ADDRESS TCB
         L     R10,TCBJSCB                GET JSCB-ADDRESS
         DROP  R10                        DELETE ADRESSABILITY
         USING IEZJSCB,R10                ADDRESS JSCB
         L     R10,JSCBPSCB               GET PSCB-ADDRESS
         DROP  R10                        DELETE ADDRESSABILITY
         USING PSCB,R10                   ADDRESS PSCB
         AUTHON                           see issue 10 page 24
         MODESET KEY=ZERO                 SET KEY ZERO
         OI    PSCBATR1,PSCBCTRL          SET OPERATOR AUTH.
         MODESET KEY=ZERO                 SET USER KEY
         AUTHOFF                          see issue 10 page 24
         EXITR                            see issue 15 pages 25-26
         LTORG
         END
THE RECEIVER
The following example shows how to drive the whole file transfer process from
the receiving installation.  From there a job is submitted to the transmitting
JES.  This job contains:
•    A step that attempts to transmit the file.
•    A job containing the ‘receive’ that will be submitted for execution to the
receiving JES from the transmitting JES in case the transmit was correctly
performed.


•    An error notification job that will be submitted for execution on the
receiving JES from the transmitting JES in case the file transmission went
wrong.
//SPD900ZZ JOB (,3142),NIPL,CLASS=G
/*XMIT NI2,DLM=ZZ
//SPD900XM JOB (,3142),NIPL,CLASS=G,MSGCLASS=T

/*ROUTE PRINT NA2/SPD900

PROFILE PROMPT
DEL '&DSN'
TSOOPER                        /* SET OPERATOR ATTRIBUTE IN BATCH  */
SET &SYSOUTTRAP = 1000
SET &RET = 0
RECEIVE USERID(&USER)          /* RECEIVE A SPECIFIC USERID        */
DATA PROMPT
 DATASET('&DSN') UNIT(WORK) SPACE(1 1) +
   TRACKS RELEASE NEW
ENDDATA
SET &SYSOUTTRAP = 0
SET &MAXLINE = &SYSOUTLINE
SET &RECCC = &RET
SET &N = 0
DO WHILE &N < &MAXLINE
 SET &N = &N + 1
 SET &SYSDVAL = &STR(&&SYSOUTLINE&N)
 SET &SYSDVAL = &STR(&SYSDVAL)
 /* THE FIRST SET SYSDVAL MOVES OVER THE CONTENTS OF SYSOUTLINEN AS
 /* ONE STRING; THE NEXT SET SYSDVAL TOKENISES THE CONTENTS OF SYSDVAL
 /* SO A READDVAL WILL GET EACH WORD INTO A SEPARATE VARIABLE
 READDVAL &A1 &A2 &A3 &A4 &A5 &A6 &A7 &A8 &A9 &AA &AB
 IF &A1 = INMR001I THEN DO
  WRITE &A1 &A2 &A3 &A4 &A5 &A6 &A7 &A8 &A9 &AA &AB
  SE '&SYSSDATE &SYSTIME &A1 &A2 &A3 &A4 &A5 &A6 &A7 &A8 &A9 &AA &AB' +
   USER(&USER) LOGON
  SET &RECEIVED = DONE
 END
 IF &A1 = INMR042I AND &RECEIVED.. = DONE. THEN DO
  WRITE POSSIBLY MORE FILES TO RECEIVE
  SE '&SYSSDATE &SYSTIME &SYSUID POSSIBLY MORE FILES TO RECEIVE' +
   USER(&USER) LOGON
 END
END
IF &RECCC > 0 AND &RECEIVED.. ^= DONE. THEN DO
 SE 'ERROR IN RECEIVE' USER(&USER) LOGON
END
ELSE DO
 SET &RECCC = 0
END
EXIT CODE(&RECCC)
