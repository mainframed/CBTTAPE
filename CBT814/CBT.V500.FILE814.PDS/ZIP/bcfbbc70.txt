JES3 exit to suppress STEPCATs
In the ICF catalog environment the use of JOBCATs and STEPCATs in JCL should be
strongly discouraged.  Below is a modification to a JES3 user exit (IATUX06)
which tests for JOBCATs or STEPCATs in a job and if either is detected fails the
job.  A suitable message is written to the JES3 log and the JESMSG dataset.  The
exit allows Technical Support jobs to run with JOBCATs or STEPCATs as both are
sometimes required, for example when setting up a back-up master catalog.  The
use of this exit means that dataset-naming conventions can be more easily
enforced through catalog protection and aliases.  This is in line with the
direction that IBM is taking towards System Managed Storage, under which JOBCATs
and STEPCATs should not be used at all.
The source for the modification is supplied in SMP format with correct sequence
numbers at the 8706 level of MVS 2.1.7 with JES3 2.1.5.

    *                       U S E R M O D                          *
    ****************************************************************
                                    */.
